# Parsed Documents

Normal ingestion writes one JSON file per document here, named with the generated document ID:

```text
doc-<uuid>.json
```

Each file serialises a `ParsedDocument` containing:

- `document`: ID, title, source path, topic, and creation time.
- `pages`: page number and extracted text for every parsed page.

This is the closest generated representation of the parser output before chunking or extraction. Newlines, PDF headers, footers, hyphenation, and layout artefacts can still be present because PyMuPDF returns the embedded text layer rather than a cleaned semantic document.

These files are ignored by Git and can be regenerated by ingesting the source again. A rerun creates a new document ID rather than replacing the old file automatically.
